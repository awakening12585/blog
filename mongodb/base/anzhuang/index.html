# MongoDB安装

  • 官网
  • 本文以mongod 4.4 版本为例

# (1)导入包管理系统使用的公钥。

  • 从终端发出以下命令以从https://www.mongodb.org/static/pgp/server-4.4.asc导入 MongoDB 公共 GPG 密钥：
wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -
//该操作应以OK.
1
2

如果您收到指示gnupg未安装的错误

  • (1)gnupg使用以下命令安装及其所需的库：
sudo apt-get install gnupg
1
  • (2)安装后，重试导入密钥：
wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -
1

# (2) 为 MongoDB 创建一个列表文件。

以下说明适用于Ubuntu 18.04 (Bionic)

echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu bionic/mongodb-org/4.4 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.4.list
1

# (3) 重新加载本地包数据库

sudo apt-get update
1

# (4) 安装 MongoDB 包

sudo apt-get install -y mongodb-org
1

# (5) 启动 MongoDB

sudo systemctl start mongod
1

如果您在启动时收到类似于以下内容的错误 mongod：Failed to start mongod.service: Unit mongod.service not found. 首先运行以下命令：

sudo systemctl daemon-reload
1

然后再次运行上面的启动命令。

# 常用命令

  • 查看状态
sudo systemctl status mongod
1
  • 停用
sudo systemctl stop mongod
1
  • 重启
sudo systemctl restart mongod
1
  • 开机自启
systemctl enable *.service
1
  • 关闭开机自启
systemctl disable *.service
1
